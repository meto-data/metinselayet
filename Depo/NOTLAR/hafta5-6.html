Ders Notları: Formlar ve Kullanıcı Girişleri

1. Formların Temel Yapısı - <form>

Kullanıcıdan veri almak için formları kullanıyoruz. Tüm girdi elemanları <form> ve </form> etiketleri arasına yazılır.

  • action : Verinin gönderileceği hedef dosyayı belirtir. (Örn: action="sonuclar.html"). Boş bırakılırsa form kendi bulunduğu sayfayı yeniler.
  • method : Verinin gönderilme şeklini belirler. GET metodu ile gönderilen veriler adres çubuğunda (? işaretinden sonra) açıkça görünür.
  • target : Linklerdeki gibi sonucun nerede açılacağını belirler. target="_blank" yeni sekmede açar.

2. Etiketleme (Label) ve Input Özellikleri

🔥 HAYATİ ÖNEMDE (name özelliği): Form verilerinin adres çubuğuna hangi değişken adıyla gideceğini belirler. Veri göndermek için mutlaka yazmak zorundayız. name yoksa veri de gitmez!
  • <label> : Form elemanlarını isimlendirir. Label'daki for değeri ile Input'taki id değeri aynı olursa birbirine bağlanırlar. Yazıya tıklayınca kutu aktifleşir.
  • required : Alanın doldurulmasını zorunlu kılar.
  • minlength / maxlength : Girilebilecek minimum ve maksimum karakter sayısını sınırlar.
  • readonly : Alanın kullanıcı tarafından değiştirilmesini engeller (Sadece okunabilir yapar).
  • size : Input alanının ekranda görünen genişliğini belirler (Varsayılan 20 karakterdir).

4. JavaScript ile URL'den Veri Okuma

Formdan GET metoduyla gönderdiğimiz verileri sonuclar.html sayfasında yakalayıp ekrana yazdırmak için şu JS yapısı kullanılır:

// 1. URL'deki parametreleri al (Soru işaretinden sonrasını okur)
const params = new URLSearchParams(window.location.search);

// 2. name özelliği ile verileri çek
const isim = params.get("isim");
const soyad = params.get("soyad");
const eposta = params.get("eposta");
const memnunluk = params.get("memnunluk");

// 3. Ekrana Yazdır (Template Literals / Backtick ` kullanarak)
document.getElementById("sonuclar").innerHTML = `
    <p>İsim: ${isim}</p>
    <p>Soyad: ${soyad}</p>
    <p>E-posta: ${eposta}</p>
    <p>Memnunluk: ${memnunluk}</p>
`;

🔥 Derse Aşkın: Sınav Odaklı Vurgular

  • Banka Soru (name Özelliği): Bir form elemanının değerinin sunucuya veya URL'ye iletilebilmesi için hangi attribute KESİNLİKLE bulunmalıdır?
    Cevap: name özelliği.
  • Browser Validation (Tarayıcı Doğrulaması): HTML5 ile arka plana (sunucuya) gerek kalmadan tarayıcının kendi kendine doğrulama yaptığı özellikler nelerdir?
    Cevap: type="email" (@ işareti kontrolü) ve required (boş bırakılamaz kontrolü).
  • action Boş Bırakılırsa: Bir formun action özelliği tanımlanmazsa form gönderildiğinde ne olur?
    Cevap: Form kendi bulunduğu sayfaya döner (sayfayı yeniler).
  • GET Metodunun Güvenlik Zafiyeti: Şifre alanlarında (type="password") karakterler ekranda yıldızlı görünse de neden GET metodu kullanılmamalıdır?
    Cevap: GET metodu veriyi adres çubuğunda (URL'de) açıkça yazarak gönderdiği için şifre herkes tarafından görünür hale gelir.
  • Dosya Sınırlama (accept): Dosya yüklerken sadece belirli uzantıların (Örn: PNG) seçilmesini sağlayan özellik nedir?
    Cevap: accept özelliği (Örn: accept="image/png").
